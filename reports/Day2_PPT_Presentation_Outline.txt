  4. Write analytical SQL queries to answer critical business questions.

--------------------------------------------------------------------------------
SLIDE 3: Data Cleaning Strategy (ETL)
--------------------------------------------------------------------------------
- Heading: Trailing Clean-Up & Outliers
- Details:
  * nav_history: Forward-filled NAV for weekends/holidays, removed duplicates, validated NAV > 0.
  * investor_transactions: Standardized transaction types (SIP, Lumpsum, Redemption) and KYC status.
  * scheme_performance: Checked return metrics and verified expense ratios fall within 0.1% to 2.5%.
  * Added derived columns (Year, Month, Alpha Categories) for quick querying.

--------------------------------------------------------------------------------
SLIDE 4: SQLite Database Schema (Star Schema)
--------------------------------------------------------------------------------
- Heading: Relational Database Architecture
- Details:
  * Dimension Tables: dim_fund (scheme specs), dim_date (calendar metrics), dim_investor (demographic details).
  * Fact Tables: fact_nav (daily valuation), fact_transactions (investor actions), fact_performance (snapshot metrics), fact_aum (bi-annual AUM).
  * Indexes built on primary keys, state, transaction type, and dates to ensure sub-millisecond query execution.
